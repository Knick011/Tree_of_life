TOL Scribe Support and Extension Use

This page describes the current pilot scope, supported workflow, and the fastest way to troubleshoot the TOL Scribe EMR Filler extension.

Current target: Juno OSCAR Rx Supported pilot browsers: Edge, Chrome, Brave, Firefox

Current pilot workflow

1. Build draft in TOL Scribe

Choose the pathway, template, or score result and copy the EMR payload from the app.

2. Open the extension

Use the extension popup on a supported Juno OSCAR Rx page and load the copied payload.

3. Fill and review

Fill the pending prescription row, then review, save, print, and sign inside the EMR.

Supported page scope

The current pilot extension is intentionally narrowed to Juno OSCAR Rx pages matching: 

https://*.junoemr.com/juno/oscarRx/*

What the extension fills

  • Prescription instructions
  • Special instruction / indication text
  • Quantity
  • Repeats
  • Comment / note to pharmacy field where mapped

What the extension does not do

  • It does not read patient charts to determine diagnosis.
  • It does not auto-sign prescriptions.
  • It does not auto-send prescriptions.
  • It does not replace clinician review inside the EMR.

Troubleshooting

  • Make sure the EMR in the TOL Scribe app header is set correctly before using Copy for EMR.
  • Confirm the active page URL is a supported Juno OSCAR Rx page.
  • If nothing fills, click Read Clipboard again in the extension and retry.
  • If the page layout changed, reload the EMR page and retry the fill action.
  • If fields are still missing, the site-specific selectors may need to be tuned for that deployment.
